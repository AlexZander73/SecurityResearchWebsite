Unsafe Browsing Agents

Agents that browse external sites can be steered by hostile pages unless tightly sandboxed.

  • Web agents
  • Sandboxing

Updated 2026-03-18

In plain words

This page explains one common AI risk in plain terms and shows a safer default you can apply quickly.

What this risk looks like

Web content is attacker-controlled. Agents that read and act on it need stronger boundaries than normal user browsing.

What can go wrong

  • Instruction hijacking via malicious page content
  • Credential leakage to lookalike domains
  • Automatic downloads or unsafe navigation loops

Safer patterns

  1. Constrain navigation domains and action types.
  2. Run browsing in isolated sandboxes.
  3. Require human approval before sensitive submissions.

Minimum control set

  • Network egress allowlists
  • Download and upload restrictions
  • Screenshot + action trace for review
AI builder reminder: Model output is not policy. Every sensitive action needs explicit guardrails.
