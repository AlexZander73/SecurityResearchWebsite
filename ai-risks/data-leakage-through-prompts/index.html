Data Leakage Through Prompts

Sensitive data can leak into prompts, logs, and outputs unless your pipeline explicitly prevents it.

  • Data protection
  • Prompt hygiene

Updated 2026-03-21

In plain words

This page explains one common AI risk in plain terms and shows a safer default you can apply quickly.

What this risk looks like

Teams may pass full records into prompts for convenience, exposing secrets and regulated data to unnecessary risk.

What can go wrong

  • PII leakage in output summaries
  • Sensitive prompts stored in logs or analytics
  • Cross-tenant context contamination

Safer patterns

  1. Minimize prompt payloads to only required fields.
  2. Mask or tokenize sensitive attributes before inference.
  3. Separate analytics from raw prompt data.

Minimum control set

  • Prompt redaction layer
  • Data classification policy for AI features
  • Output scanning for sensitive patterns
AI builder reminder: Model output is not policy. Every sensitive action needs explicit guardrails.
